## 1. 项目现状

### 1.1 当前基础

| 维度 | 现状 |
|------|------|
| **目标平台** | Paper 26.1+（Minecraft Java 版） |
| **语言** | Java 25+，Gradle（Kotlin DSL）构建 |
| **当前发布渠道** | [Hangar](https://hangar.papermc.io/OrzMC/OrzMC)（免费） |
| **许可证** | ❌ 无许可证文件 → 默认 "All Rights Reserved" |
| **集成平台** | QQ（NapCatQQ/OneBot）、Discord（JDA）、飞书（Webhook） |
| **代码质量** | 分层架构、单元测试、集成测试、CI/CD、文档完整 |
| **用户群** | 私服 `orzmc.jokerhub.cn` 实际运行 |

### 1.2 商业化的核心优势

- **功能成熟** — 不是玩具项目，有完整测试和架构
- **多平台机器人集成** — QQ + Discord + 飞书，**极少有插件同时支持三者**
- **中文市场独特性** — 海外插件没有 QQ/飞书集成，这是你的独占蓝海
- **代码质量好** — 分层架构、测试覆盖、文档健全，商业化后维护成本可控

### 1.3 当前短板（需补齐）

- 缺少开源许可证文件
- 没有付费版构建通道
- 没有官网或产品页面
- 缺少用户社区（QQ群、Discord）

## 2. 平台对比

### 2.1 付费分发自营平台

| 平台 | 付费支持 | 佣金 | 许可证系统 | 开发者门槛 | 中文流量 | 推荐度 |
|------|---------|------|-----------|-----------|---------|:------:|

| **[Polymart](https://polymart.org)** | ✅ 原生 | 15% | ✅ 内置密钥+自动更新 | 低 | 低 | ⭐⭐⭐⭐ |
| **SpigotMC Premium** | ✅ 原生 | **40%** | ✅ 内置 | 中（需审核） | 高 | ⭐⭐⭐ |

### 2.2 各平台详解

#### BuiltByBit（前 MC-Market）— ⭐⭐⭐⭐⭐ 首选

- **流量**：Minecraft 付费插件最大市场，日活买家量大
- **许可证**：内置完整系统 — 买购买后自动生成密钥，插件通过 API 在线验证
- **佣金**：标准 10%，或 $5/月订阅费 → **全免佣金**
- **门槛**：$5 一次性验证费
- **社区**：有买家评价、论坛讨论、技术支持区
- **建议**：付费版首发地

#### Polymart — ⭐⭐⭐⭐ 备选

- **界面**：现代而简洁，用户体验好
- **特性**：支持自动更新推送，买家购买后自动获得新版本
- **佣金**：15%
- **建议**：作为第二渠道扩展曝光

#### SpigotMC Premium — ⚠️ 要慎重

- **流量**：最大，但佣金最高达 **40%**
- **审核**：提交严格，审核周期长
- **建议**：等付费版在 BuiltByBit 有销量和评价后，再考虑上架

### 2.3 免费版保留渠道

即使推出付费版，建议继续在以下平台维护**功能受限的免费版**作为引流：

- **Hangar** — 保持当前免费版
- **GitHub Releases** — 提供开源免费版
- **Modrinth** — 可选的额外曝光

### 3.2 其他可选模式

| 模式 | 说明 | 适用性 |
|------|------|--------|
| **一次性付费** | 付费下载，终生使用 | ⭐⭐⭐ 主流方案，最易接受 |
| **订阅制** | $4.99-9.99/月，含持续更新+支持 | 适合功能持续迭代的插件 |

| **SaaS 托管** | 托管式后台 + 机器人 | 适合长远发展，短期不做 |

## 4. 定价策略

### 4.1 海外定价参考区间

| 插件类型 | 价格区间 | 例子 |
|---------|---------|------|
| 单功能小插件 | $5-10 | 记分板、公告插件 |
| 中大型管理插件 | **$15-25** | Lands ($20), SuperiorSkyblock ($15) |
| 全功能套件 | $30-50 | AdvancedEnchantments ($35), Oraxen ($20) |
| 订阅制 | $5-10/月 | 含持续更新+支持 |

**→ OrzMC 建议海外首价：$19.99（一次性定价）**

### 4.2 国内定价建议（人民币）

| 版本 | 价格 | 说明 |
|------|:----:|------|
| 个人版 | ¥39-69 | 服务器 ≤ 50 人 |
| 商业版 | ¥99-199 | 无人数限制 |
| 技术支持 | ¥299+ | 含配置指导 + 定制开发 |

### 4.3 定价原则

- **首发不宜过高**：初期以低价或折扣积累用户评价（Launch 折扣 -30%）
- **留涨价空间**：定价 $19.99，未来功能扩充后可涨至 $24.99-29.99
- **区分明显**：免费版与付费版功能差异要清晰，让用户觉得付费值得
- **捆绑折扣**：考虑 "买 2 份减 10%" 等批量优惠（大型服务器可能需要多服授权）

## 5. 防盗版方案

付费插件的核心挑战是盗版。建议分层防护：

### 5.1 第一层：平台内置验证（必需）

BuiltByBit 和 Polymart 都提供内置许可证系统：
- 买家购买后收到唯一激活密钥
- 插件启动时通过 HTTP API 向平台验证密钥有效性
- 平台已拦截大部分简单盗版
- **投入：零开发成本**

### 5.2 第二层：自有验证服务器（推荐）

```
插件启动 → 读取许可证密钥 → HTTPS 请求你的验证服务器
  → 服务器解密校验签名
  → 返回：有效 / 无效 / 超过绑定数
  → 无效响应 → 插件行为：

       ┌─ 1 小时内：仅在控制台输出警告
       ├─ 1-24 小时：每次管理员 OP 登录时提醒
       ├─ 24-72 小时：禁用高级功能（保留基础功能保服务器）
       └─ 72 小时+：完全禁用但保留 /orz 帮助命令
```

**实现方案选择：**

| 方案 | 难度 | 成本 | 推荐度 |
|------|:----:|:----:|:------:|
| 自建轻量 API（Java/Sprint Boot） | 中 | 服务器费用 | ⭐⭐⭐⭐ |
| 自建轻量 API（Node.js/Cloudflare Worker） | 低 | 几乎免费 | ⭐⭐⭐⭐⭐ |
| Lunr | 低 | 免费起步 | ⭐⭐⭐ |
| KeyAuth | 低 | 免费起步 | ⭐⭐⭐ |

**独立验证服务器的优势**：即使平台方被攻破，你的自有验证仍然安全。

### 5.3 第三层：代码混淆（建议）

使用 **ProGuard** 或 **yGuard** 在构建时混淆代码：

```kotlin
// build.gradle.kts 示例 — ProGuard 集成
plugins {
    id("com.guardsquare.proguard") version "7.x"
}

tasks.register<ProguardTask>("obfuscate") {
    configuration("proguard.pro")
    injars(tasks.shadowJar.flatMap { it.archiveFile })
    outjars(layout.buildDirectory.file("libs/orzmc-premium-obfuscated.jar"))
    // libjars 包含 PaperAPI 等公共库（不混淆）
}
```

**关键规则：**
- ✅ 混淆你的业务逻辑包（`com.jokerhub.paper.plugin.orzmc.features.*`）
- ✅ 混淆基础设施包（`com.jokerhub.paper.plugin.orzmc.infra.*`）
- ❌ **不要混淆** PaperAPI 的公开类和方法
- ❌ **不要混淆** 插件入口类（`OrzMC`）和 `plugin.yml` 中的引用

### 5.4 第四层：服务器绑定（可选）

- 每个许可证绑定最多 3 个服务器的 IP 或玩家 UUID
- 防止一份购买被无限服务器使用
- 用户可在后台管理绑定（解绑/更换）

### 5.5 对抗盗版的底线思维

> **不存在 100% 防破解的方案。** Someone will always crack it.
>
> 目标不是让破解不可能，而是让：
> 1. 破解成本 > 购买成本
> 2. 破解版本不稳定（加校验、加延迟、加随机检查）
> 3. 普通人找不到/不敢用破解版

## 7. 法律与许可

### 7.1 当前状态

此仓库目前**没有许可证文件**，根据国际版权法，默认为 **All Rights Reserved**。
- ✅ 你可以随时闭源商业化
- ✅ 你可以出售插件
- ⚠️ 用户也没有任何明确的使用许可
- **建议：尽快补充许可证文件**

### 7.2 推荐许可策略

| 版本 | 许可证 | 说明 |
|------|--------|------|
| GitHub/Hangar 免费版 | **GNU GPL v3** | 可自由使用、修改、分发，但必须开源 |
| Premium 付费版 | **闭源**（仅二进制 JAR） | 通过购买获得使用许可 |
| Enterprise 企业版 | **自定义商业许可** | 附源代码查看权，但无再分发权 |

**为什么免费版选 GPL v3 而不是 MIT？**
- MIT 允许任意使用，包括别人把你的代码打包成商业产品卖
- GPL v3 要求衍生作品也必须开源，有效保护你的商业版代码
- GPL v3 的传染性也为付费版提供了竞争优势（想闭源集成 → 买商业版）

### 7.3 PaperAPI 的许可

- Paper 使用 **MIT 许可证**，完整允许商业使用
- 你的插件只是依赖 PaperAPI，不是修改 Paper
- **没有任何法律障碍**

### 7.4 需要准备的法律文件

- [ ] 免费版开源许可证文件（LICENSE.md）
- [ ] Premium 版 EULA（最终用户许可协议）
- [ ] 隐私条款（验证服务器会收集 IP、服务器信息）
- [ ] 退款政策（BuiltByBit 自动处理，自建渠道需自行制定）

### 7.5 许可验证建议

在付费版中加入版权声明和许可验证：
```
plugin.yml 中添加：
  website: https://your-site.com
  description: OrzMC Premium - 授权给 [server-name]，许可证号 [license-key]
```

## 9. 分阶段路线图

### Phase 1 — 准备期（1-2 周）

```
目标：完成商业化基础设施搭建

  ├─ Gradle 构建配置：免费版 / 付费版双通道
  │   ├─ free flavor：浅层功能开关
  │   └─ premium flavor：完整功能 + 许可证验证
  ├─ 内置许可证验证模块
  └─ ProGuard 混淆配置

□ 社区建立
  ├─ 创建 QQ付费用户群 / Discord 服务器

□ 物料准备
  ├─ BuiltByBit 产品页面文案（中英双语）
  ├─ 截图 + 功能展示图（8-10 张）
  └─ 功能对比表（免费版 vs 付费版）
```

```
目标：在 BuiltByBit 首发付费版

  └─ 上架付费版

  ├─ 上市首周 30% 折扣 ($19.99 → $13.99)

  └─ 在 SpigotMC 免费版帖发布更新通知

□ 内容发布
  ├─ B站发布简介视频（中文字幕/配音）
  ├─ MCBBS/MineBBS 发布宣传帖
  └─ 知乎发布技术文章软推广
```

### Phase 3 — 增长期（1-3 个月）

```
目标：稳定收入 + 用户增长

□ 渠道扩展
  ├─ 在 Polymart 上架（作为第二渠道）

  ├─ 销量统计（按平台、按地区）

  └─ 用户来源分析（哪条渠道带来最多客户）
```

```
目标：收入多元化 + 品牌建立

□ 产品线扩展
  ├─ 推出订阅版（$4.99/月，含持续更新+支持）

  └─ 考虑参加 Minecraft 开发者社区活动

  ├─ 评估 SaaS 模式（托管式机器人后台）
```

### 9.1 关键检查点

| P4 | 月收入 ≥ $1,000 | 考虑全职投入或扩大团队 |

### 10.1 Gradle 免费版/付费版构建参考

```kotlin
// build.gradle.kts  — 双通道构建
val isPremium = project.hasProperty("premium")

java {
    // 根据 premium 属性决定是否包含扩展功能
}

tasks {
    // 免费版构建
    register<Jar>("buildFree") {
        archiveFileName.set("OrzMC-free-${project.version}.jar")
        from(sourceSets.main.get().output)
        // 排除付费功能资源
    }

    // 付费版构建
    register<Jar>("buildPremium") {
        archiveFileName.set("OrzMC-premium-${project.version}.jar")
        from(sourceSets.main.get().output)
        // 包含许可证验证代码
        // 执行 ProGuard 混淆
    }
}
```

更推荐的方式：在 Java 代码中通过系统属性或编译常量开关功能：
```java
// FeatureFlags.java
public enum FeatureFlags {
    TNT_PROTECTION("premium"),
    GEO_IP("premium"),
    CROSS_SERVER_PORTAL("premium"),
    TELEPORT_BOW("premium");

    private final String tier;

    public boolean isAvailable() {
        return !"premium".equals(tier) || OrzMC.getInstance().isPremium();
    }
}
```

### 10.2 验证服务器最低实现

```java
// 内置验证客户端 — 启动时调用
public class LicenseValidator {
    private static final String VERIFY_URL = "https://verify.orz-mc.com/check";

    public static LicenseResult validate(String licenseKey, String serverIp) {
        // 1. 构建请求（包含密钥、服务器 IP、插件版本）
        // 2. 发送 HTTPS POST 到验证服务器
        // 3. 服务器解密签名、检测黑名单
        // 4. 返回 { status: "valid"|"invalid"|"banned", expiresAt: "..." }
        // 5. 本地缓存结果（有效期 24 小时，防止验证服务器宕机导致插件不可用）
    }
}
```

验证服务器推荐技术栈：
- **方案 A**：Cloudflare Workers + KV（几乎免费，全球低延迟）
- **方案 B**：Vercel Edge Function + Upstash Redis（同样低成本）  
- **方案 C**：阿里云函数计算（国内用户延迟最低）
